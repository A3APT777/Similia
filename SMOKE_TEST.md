# Smoke Test — перед каждым деплоем

Быстрая проверка: открой каждый пункт, убедись что работает. Занимает ~5 минут.

## 🔐 Авторизация
- [ ] Открыть `/login` → войти → попасть на dashboard
- [ ] Открыть `/register` → форма отображается без ошибок

## 👤 Пациенты
- [ ] Dashboard → список пациентов отображается
- [ ] Кнопка «Новый пациент» → форма открывается, дропдаун типа конституции работает (z-index)
- [ ] Создать тестового пациента → перенаправление на карточку

## 📋 Консультации
- [ ] Открыть карточку пациента → кнопка «Новый приём» создаёт консультацию
- [ ] Редактор консультации → поля жалоб заполняются и автосохраняются
- [ ] Переключить тип на «Острый» → форма жалоб меняется под острый случай
- [ ] Кнопка «Реперторий» → открывает мини-реперторий внутри консультации

## 📖 Реперторий
- [ ] Открыть `/repertory` → поиск «головная боль» → появляются рубрики
- [ ] Кликнуть на рубрику → раскрывается список препаратов
- [ ] Нажать [+] → рубрика добавляется в Анализ справа
- [ ] Кнопка «Обучение» → панель туториала открывается

## 📩 Анкеты (публичные ссылки)
- [ ] Карточка пациента → «Отправить анкету»:
  - Если первичной анкеты нет → ссылка на первичную
  - Если есть → ссылка на острый случай

## ✅ Финальный шаг
- [ ] `npx next build` — без ошибок TypeScript

## Как использовать

1. Перед деплоем — пробежать весь список
2. Если что-то сломано — зафиксировать в задаче, починить до деплоя
3. Если добавили новую функцию — добавить её в чеклист

> **Правило:** если деплоишь и не проверил — сам виноват 😄
