# Requirements — Similia v1.1 Onboarding

## v1.1 Requirements

### Этап 2 — Карточка пациента

- [ ] **TOUR2-01**: После создания пациента (конец этапа 1) пользователь видит предложение продолжить обучение — начать тур по карточке
- [ ] **TOUR2-02**: Тур объясняет кнопку «Пригласить пациента» (IntakeLinkButton) — как отправить анкету новому пациенту
- [ ] **TOUR2-03**: Тур объясняет кнопку «Записать на приём» (ScheduleButton) — как создать консультацию с датой и временем
- [ ] **TOUR2-04**: После записи на приём тур продолжается в редакторе консультации (автоматически или по клику на карточку приёма)

### Этап 2 — Редактор консультации

- [ ] **TOUR2-05**: Тур показывает поле жалоб и объясняет что туда писать
- [ ] **TOUR2-06**: Тур показывает кнопку открытия мини-репертория в тулбаре
- [ ] **TOUR2-07**: Тур ведёт по мини-реперторию: поиск рубрики → добавление в анализ → топ препаратов → назначение препарата
- [ ] **TOUR2-08**: После назначения из репертория тур показывает InlineRx (поле назначения) с уже заполненным препаратом
- [ ] **TOUR2-09**: Тур показывает кнопку «Завершить приём» и объясняет что происходит после нажатия
- [ ] **TOUR2-10**: По завершении этапа 2 показывается тост-поздравление и предложение перейти к туру по реперторию

### Этап 3 — Основной реперторий

- [ ] **TOUR3-01**: Тур запускается на странице `/repertory` — автоматически или по приглашению после этапа 2
- [ ] **TOUR3-02**: Тур показывает поле поиска и объясняет как искать рубрики (на русском и английском)
- [ ] **TOUR3-03**: Тур показывает список рубрик с грейдами препаратов и объясняет что они означают
- [ ] **TOUR3-04**: Тур объясняет добавление рубрики в анализ и кнопку элиминации (E)
- [ ] **TOUR3-05**: Тур показывает панель анализа с топ препаратами и финальным счётом
- [ ] **TOUR3-06**: По завершении тура показывается финальное поздравление — онбординг завершён

### Верификация

- [ ] **VERIFY-01**: Каждый шаг тура указывает на реально существующий элемент (`data-tour` атрибут присутствует в DOM)
- [ ] **VERIFY-02**: Описание в туре соответствует реальному поведению элемента
- [ ] **VERIFY-03**: Все `data-tour` атрибуты добавлены во все нужные компоненты

## v2 Requirements (отложено)

- Возможность перезапустить любой этап тура из настроек
- Анимированные подсказки с gif/видео

## Out of Scope

- Видео-туториалы — слишком сложно для v1.1
- A/B тестирование онбординга — не нужно сейчас
