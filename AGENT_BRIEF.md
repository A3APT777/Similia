# Similia — брифинг для агентов-аудиторов

## Что это
Similia (simillia.ru / homeocase.vercel.app) — цифровой кабинет для врачей-гомеопатов. MVP-стадия, один разработчик (начинающий).

## Стек
- Next.js 16 (App Router, Turbopack), React 19, TypeScript strict
- Supabase (PostgreSQL + Auth + Storage), RLS на всех таблицах
- Tailwind 4, framer-motion, driver.js (legacy, заменяется кастомными обучалками)
- Деплой: Vercel (регион fra1)
- Домены: simillia.ru (основной), homeocase.vercel.app (алиас)

## Целевая аудитория
Практикующие гомеопаты в РФ и СНГ. ~5000-10000 человек. Основной язык — русский. Английский — вторичный (i18n покрытие ~50%).

## Ключевые решения (уже приняты)
- Реперторий Кента (Repertorium Publicum, 74 482 рубрики) — встроен в консультацию (мини) и отдельной страницей (полный)
- Грейды 1/2/3 по Кенту: grade 3 = жирный, grade 2 = курсив, grade 1 = обычный
- Формула подсчёта: балл = Σ(грейд × вес) по всем рубрикам
- Элиминация: исключает препараты отсутствующие в отмеченных рубриках
- Два типа консультации: хроническая и острая (разные поля)
- Потенции: C-шкала (6C-10M) + LM-шкала (LM1-LM6) + кастомное поле
- Форма приёма: чипы (Сухая доза / Раствор / Ольфакция / Однократно / Ежедневно / По необходимости)
- Автосохранение: debounce 1.5сек с exponential backoff при ошибках
- Обучалки: кастомная система (панель + overlay + glow), НЕ driver.js
- Тур по сайту: 5 шагов + контекстные подсказки (FirstTimeHint)

## Известные проблемы (техдолг)
- i18n: 770 хардкодных кириллических строк в 49 файлах. При EN — микс языков. Отложено до выхода на международный рынок.
- CSP содержит unsafe-inline (nonce не реализован)
- Rate limiting in-memory (бесполезен на serverless). Нужен Upstash Redis.
- Тесты: 1 файл из 100+ модулей
- framer-motion в бандле (~150KB) — возможно избыточен

## Терминология (глоссарий)
- **Консультация** — запись о приёме в системе
- **Приём** — событие в расписании
- **Препарат** — гомеопатическое средство (не «назначение», не «remedy» в UI)
- **Гранулы** — форма приёма (не «горошинки»)
- **Рубрика** — симптом в реперториуме Кента
- **Грейд** — степень подтверждённости препарата при симптоме (1-3)
- **Вес** — важность рубрики для конкретного пациента (1-3)
- **Элиминация** — фильтр: оставить только препараты присутствующие в рубрике
- **Симилимум** — наиболее подходящий препарат (кандидат, не окончательный ответ)
- **Реперторий** — основной термин (не «реперториум» в UI)

## Стандарты кода
- Шрифты: минимум 12px
- Touch-таргеты: минимум 24px (w-6 h-6), идеально 44px
- Контрастность: WCAG AA (4.5:1 для текста)
- aria-label на всех интерактивных элементах
- role="dialog" + aria-modal на модалах, focus trap + Escape
- Zod-валидация на всех server actions
- Обращение: на «вы» (медицинский SaaS)
- CSS: предпочтительно CSS-переменные (--sim-*), не хардкод hex
